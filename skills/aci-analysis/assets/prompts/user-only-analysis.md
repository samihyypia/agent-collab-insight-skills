# User-Only 定性分析 Prompt

> 将此 Prompt 与 `.aci/<source>-YYYYMMDDHHmm/01-数据归一化/user-only/` 下的结构化 JSON，以及 `03-分析材料包/` 中的相关 packet 一起交给 AI，生成中文定性分析报告。输入数据仅包含用户输入消息，不包含 AI 回答正文。定量统计由代码生成，本 Prompt 只负责定性洞察。

## 输出语言约束

最终报告必须使用中文。元数据、代码、路径、命令、字段名和原始引用可以保留原文。

## 输入

你将收到以下数据：

1. **全局模式**：所有会话的 user-only JSON 与全局 packet。
2. **项目模式**：某个项目 packet，以及相关 user-only JSON。
3. **典型会话模式**：选取的代表性会话 packet，以及对应 user-only JSON。

数据结构概览：

- `session_id`, `session_name`, `source_type`, `model`, `created_at`：会话元信息。
- `project_name`, `project_dir`：项目归属信息。
- `turns[]`：仅包含用户消息的数组。
- `turns[].turn_index`：轮次序号。
- `turns[].user_message.text`：用户输入文本。
- `turns[].user_message.slash_command`：斜杠命令或 skill 命令，如有。

## 任务

基于用户消息进行定性分析。定量统计数据已由代码生成，不要重复堆砌数字，要聚焦数据背后的洞察。

### 分析维度

1. **用户意图图谱**：用户在做什么，核心工作主题是什么。
2. **决策风格分析**：用户如何做决策，是快速选择、详细讨论、主动提出还是被动响应。
3. **协作模式画像**：用户偏好的协作方式，例如 Skill 驱动、渐进式、批量式或探索式。
4. **需求演进轨迹**：需求如何在会话内和跨会话间演进。
5. **痛点与摩擦点**：反复出现的挫折、纠正、返工或效率瓶颈。
6. **用户成熟度评估**：用户对 AI 编程助手的掌控程度。
7. **改进建议**：给出可操作建议，面向用户和工具各 3-5 条。

### 分析原则

- **只看用户说了什么**：不推测 AI 回答正文。
- **跨会话关联**：识别同一项目或相似任务之间的连续性。
- **以用户为中心**：目标是理解用户，而不是评价 AI。
- **引用证据**：关键结论引用具体会话 ID 和轮次。

## 输出格式

按 packet 指定的目标路径写出中文 Markdown 报告。报告标题应区分全局、项目或典型会话。
